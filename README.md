# Waseet (وسيط)

Private (approval-only) B2B real-estate marketplace for **Saudi Arabia**, connecting
verified property **developers** with licensed **realtors**, with commission tracking
and an admin approval/console. Three roles: **Admin**, **Developer**, **Realtor**.

## Repository layout

| Folder | What it is |
| --- | --- |
| [`waseet-react/`](waseet-react/) | Frontend — React 18 + Vite + React Router. Marketplace, portals, admin. |
| [`waseet-backend/`](waseet-backend/) | API — Express + Prisma (PostgreSQL) + Redis + S3-compatible storage (RustFS). |
| [`waseet-infra/`](waseet-infra/) | Local Docker infra — Postgres, Redis, object storage, pgweb DB UI. |

## Run locally

```bash
# 1. Infrastructure
cd waseet-infra && docker compose up -d

# 2. Backend  (http://localhost:19000)
cd waseet-backend && npm install
npx prisma db push
npm run seed                 # demo admin / developer / realtor + Saudi projects
node --max-old-space-size=4096 --env-file=.env prisma/importGeo.js   # Saudi cities + districts
npm run dev

# 3. Frontend (http://localhost:5173)
cd waseet-react && npm install && npm run dev
```

Demo logins and service URLs are in [`LOGINS.md`](LOGINS.md).

## Deploy to a VPS (one command)

The whole app runs as Docker containers — one per service (postgres · redis · rustfs storage · backend · frontend) in a single [`docker-compose.yml`](docker-compose.yml). First-time setup:

```bash
git clone <this-repo> waseet && cd waseet
bash install.sh          # installs Docker if needed, asks a few questions, boots everything
```

`install.sh` is a guided installer: it generates all secrets, asks for the admin email/password, whether to load demo data, and (optionally) a domain — which it wires up with **automatic HTTPS** (nginx vhost + certbot) **without touching any other sites** already on the server. It also schedules nightly backups.

```bash
bash update.sh                       # pull latest code — takes a mandatory backup first, no data loss
bash backup.sh create                # full backup (DB + storage) → ./backups/
bash backup.sh list
bash backup.sh restore <file>        # restore DB + storage from a backup
```

Only the web container is published (to `127.0.0.1:APP_PORT`); the database, cache and storage stay on the internal Docker network. The production `.env` (generated by `install.sh`) is git-ignored.

> **Security note:** `.env` files are committed to this private repo for convenience.
> They hold local development secrets (DB / JWT / SMTP) — **rotate them before any
> production deployment** and never make this repository public as-is.
